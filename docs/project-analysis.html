📌 প্রজেক্ট সারাংশ Overview

ZombieCoder Agent Module v2.0.0 একটি মাল্টি-এজেন্ট AI সিস্টেম যা ৫টি ভিন্ন বিশেষায়িত AI এজেন্ট নিয়ে কাজ করে। এটি Vercel AI SDK v6 এবং OpenCode Zen ফ্রি মডেলের উপর ভিত্তি করে তৈরি। প্রতিটি এজেন্টের আলাদা ভূমিকা, system prompt এবং টুল পারমিশন রয়েছে।

টেক স্ট্যাক:

TypeScript 5.9 Node.js 20+ Express.js 5 Vercel AI SDK v6 SQLite (better-sqlite3) LangChain Zod Validation

🔴 ৩. ক্রিটিক্যাল সমস্যা (অবিলম্বে সমাধান প্রয়োজন)

⚠️ C-1: স্পেশালিস্ট এজেন্ট ফাইল নেই — স্ক্রিপ্ট ভাঙ্গা CRITICAL

সমস্যা: scripts/run_agent_checks.cjs, run_agent_checks.mjs, এবং run_agent_checks_dist.mjs — তিনটি স্ক্রিপ্টই src/agents/specialists/SolutionArchitectAgent.js ইত্যাদি থেকে import করে, কিন্তু এই ফাইলগুলোর অস্তিত্ব নেই।

প্রভাব: স্ক্রিপ্ট রান করলে MODULE_NOT_FOUND error হবে।

সমাধান: স্ক্রিপ্টগুলো আপডেট করে agentFactory.ts ব্যবহার করতে হবে, অথবা স্ক্রিপ্টগুলো মুছে ফেলতে হবে।

// পরিবর্তে এইভাবে import করা উচিত:
import { createAgent } from '../src/agents/agentFactory.js';
const agent = createAgent('solution-architect');

🔴 Impact: High

⚠️ C-2: src/config/identity.ts মিসিং — ডক্সে ভুল রেফারেন্স CRITICAL

সমস্যা: docs/index.html এবং README.md এ src/config/identity.ts রেফারেন্স করা আছে, কিন্তু এই ফাইলটি বিদ্যমান নেই। আইডেন্টিটি এখন src/agents/agentFactory.ts এ সংজ্ঞায়িত। এছাড়াও tsconfig.json এ "include": ["config/**/*"] আছে কিন্তু config ডিরেক্টরি খালি।

সমাধান: ডক্স আপডেট করে সঠিক পাথ উল্লেখ করতে হবে, এবং tsconfig.json থেকে config include সরাতে হবে (বা config ডিরেক্টরি ব্যবহার করতে হবে)।

🔴 Impact: High

⚠️ C-5: AgentLoop-এ System Prompt ডুপ্লিকেশন CRITICAL

সমস্যা: src/agents/agentFactory.ts ইতিমধ্যে টুল ইনফর্মেশন system prompt-এ inject করে। কিন্তু src/agents/core/AgentLoop.ts আবার একই টুল ইনফর্মেশন system prompt-এর সাথে append করে। ফলে LLM-এ ডুপ্লিকেট টুল তথ্য যায়।

// agentFactory.ts — ইতিমধ্যে inject করে
const fullSystemPrompt = [
  getIdentityPrefix(),
  personaText,
  workflowText,
  Object.keys(tools).length > 0
    ? `\nAVAILABLE TOOLS:\n...`  // ← এই অংশ
    : '',
].join('\n\n');

// AgentLoop.ts — আবার append করে
const fullSystemPrompt = `$\{this.config.systemPrompt}$\{toolBlock}\n\n...`;  // ← আবার!

সমাধান: AgentLoop থেকে toolBlock অপসারণ করতে হবে, কারণ agentFactory ইতিমধ্যে এটি inject করে।

🟡 Impact: Medium

⚠️ C-6: Windows-নির্ভর কোড server.ts-এ CRITICAL

সমস্যা: server.ts এর শুরুতে আছে:

try { const { execSync } = await import('child_process');
      execSync('chcp 65001', { stdio: 'ignore' }); } catch {}

এই chcp 65001 কমান্ডটি শুধুমাত্র Windows কমান্ড প্রম্পটে কাজ করে। Linux/macOS-এ এটি silent fail করলেও, এটি অপ্রয়োজনীয় এবং কনফিউশন তৈরি করে।

সমাধান: প্ল্যাটফর্ম চেক যোগ করা (process.platform === 'win32') অথবা পুরোপুরি সরিয়ে ফেলা।

🟡 Impact: Low

⚠️ C-7: tsconfig.json-এ noImplicitAny: false CRITICAL

সমস্যা: tsconfig.json এ "noImplicitAny": false সেট করা আছে। এর মানে TypeScript এমন অনেক টাইপ চেক করবে না যা implicit any টাইপ পায়। এটি টাইপ সেফটি দুর্বল করে এবং runtime error-এর সম্ভাবনা বাড়ায়।

সমাধান: noImplicitAny: true সেট করে সব implicit any ঠিক করতে হবে।

🟡 Impact: Medium

⚠️ C-8: .env ফাইলে API Key প্লেইনটেক্সট CRITICAL

সমস্যা: .env ফাইলে API Key প্লেইনটেক্সটে সংরক্ষিত আছে। যদিও .env ফাইলটি .gitignore-এ আছে, তবুও লোকাল মেশিনে এটি একটি নিরাপত্তা ঝুঁকি।

সমাধান: একটি .env.example ফাইল তৈরি করা উচিত যেখানে placeholder API Key থাকবে। প্রকৃত .env ফাইল কখনই শেয়ার করা উচিত নয়।

🟡 Impact: Medium (if .env is not shared)

⚠️ C-9: টেস্ট কভারেজ শূন্য CRITICAL

সমস্যা: package.json এ test script আছে: "test": "echo \"No tests yet\" && exit 0"। কোনো unit test, integration test, বা e2e test নেই।

সমাধান: Vitest বা Jest ব্যবহার করে টেস্ট ফ্রেমওয়ার্ক সেটআপ করতে হবে এবং প্রতিটি মডিউলের জন্য টেস্ট লিখতে হবে।

🔴 Impact: High

⚠️ C-10: utils/mathUtils.ts — অপ্রাসঙ্গিক কোড CRITICAL

সমস্যা: utils/mathUtils.ts ফাইলটি GCD (Greatest Common Divisor) এবং LCM (Least Common Multiple) ক্যালকুলেট করে। এই ফাইলটি ZombieCoder AI Agent সিস্টেমের সাথে সম্পর্কিত নয়। এটি সম্ভবত অন্য কোনো প্রজেক্ট থেকে গেছে।

সমাধান: ফাইলটি সরিয়ে ফেলা বা আলাদা একটি ইউটিলিটি প্যাকেজে স্থানান্তর করা।

🟡 Impact: Low

❌ উন্নতির প্রয়োজন Risk
  • CORS সব origin-এর জন্য open — production-এ সীমিত করা উচিত
  • কোনো rate limiting নেই — DoS attack সম্ভব
  • কোনো request body validation নেই (শুধু size limit)
  • SQL injection risk কম কিন্তু prepared statement ব্যবহার করা ভালো
  • API key .env-এ plaintext — এনক্রিপ্ট করা উচিত
  • HTTPS নেই — production-এ SSL/TLS প্রয়োজন
🛡️ নিরাপত্তা সুপারিশ Recommendations
  1. CORS সীমিত করুন: origin: process.env.CORS_ORIGIN || 'http://localhost:3000'
  2. Rate Limiting যোগ করুন: express-rate-limit প্যাকেজ ব্যবহার করে
  3. Input Validation: সব request body Zod schema দিয়ে validate করুন
  4. Helmet.js: HTTP security headers সেট করতে
  5. API Key Encryption: .env ফাইল এনক্রিপ্টেড রাখুন বা vault service ব্যবহার করুন
  6. Graceful Shutdown: DB connection properly close করুন
